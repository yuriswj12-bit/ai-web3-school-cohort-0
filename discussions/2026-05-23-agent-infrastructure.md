# 讨论记录：Agent 基础设施（2026-05-23）

**参与者**：hek + Lumo  
**起点**：Sophia 会议纪要中的 ERC-8004、x402、reputation、CROPS

## hek 核心观点

1. AI agent 目前真实经济规模仍小，称其为「经济参与者」仍勉强。
2. Web3 目前依附 Web2 发展，真正去中心化存疑（暂按下不论）。
3. 交易、协商、支付、信誉四个需求中，支付（x402）和信誉（ERC-8004）相对清晰，其余仍不成熟。
4. 去年曾有一个信誉类项目（疑似 Ethos 相关）火过但商业化失败，存活仅一个多月。
5. Web3 独特瓶颈不是能力，而是 agent 与钱包之间的安全边界。

## 关键概念澄清（已理解）

- **Reputation**：agent 过去任务的链上信用记录（成功率、被 front-run 次数等）。
- **Oracle**：把链下/链上历史数据带给合约的桥梁。reputation oracle 即提供 agent 信誉数据的服务。
- **Front-run**：交易在 mempool 中被更高 gas 的交易插队获利。
- **x402**：机器对机器的 HTTP 支付状态码，支持稳定币直接支付。
- **ERC-8004**：agent 身份 + 注册表 + 信誉标准，目前仍早期。
