## 核心观点

AI agents 正在从工具变成经济参与者。它们需要中立的基础设施来交易、协商、支付和建立信誉。

Ethereum 正在成为这个协调层。它的价值在于提供去中心化结算和承诺执行，而非由单一公司控制。

## 关键框架

CROPS 原则决定 Ethereum 的独特性：
- Censorship resistance：抗审查
- Open source and free：开源自由
- Privacy：隐私（零知识证明）
- Security：安全

这些原则支持身份、声誉、支付和可执行协议。

## 具体标准与工具

- ERC-8004：为 agents 提供可验证身份、注册表和声誉系统。任何接触点都能查看信任状态。
- x402：机器对机器支付状态码，支持稳定币直接支付，无需信用卡或 API key。

## 真实案例

Ethereum Foundation 试点项目：AI agents 辅助分配公共物品资金。人类设定规则和护栏，agents 扩展判断规模。

## 应用方向

支付是最直接的落地场景。通过智能合约设置明确边界（例如只允许在特定范围内消费），避免 prompt 注入风险。

## Q&A 重点

- Reputation 属于 agent 本身，而非背后人类。可扩展到 API、oracle 等数字实体。
- 应用比新协议更重要。熟悉本地市场的 builder 有优势。
- 协议层无法覆盖所有 job，应用层需要定制。
- 最终由 builder 决定未来基础设施是中立开放还是中心化控制。
